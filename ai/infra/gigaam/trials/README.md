# GigaAM — эксперимент по замене платной транскрипции

Изолированный стенд для проверки локального ASR (GigaAM) на CPU перед тем, как
строить serving-враппер. **Самодостаточный**: собственный `docker-compose.yml`,
к основному стеку проекта НЕ подключён и подключаться не должен.

## Зачем

Альтернатива транскрипции — платный Whisper API. Цель эксперимента была в том,
чтобы заменить его на self-hosted GigaAM-v3 (лучший русский) и перестать
переплачивать. Эксперимент завершён в пользу GigaAM: рабочий сервис лежит
в [../](..) и подключён к LiteLLM как `transcription-gigaam`
(см. [infra/litellm/config.yaml](../../../../infra/litellm/config.yaml)).

Этот шаг — только **замер**: тянет ли GigaAM CPU на ненагруженной машине и
устраивает ли качество русского. Враппер пишем уже по цифрам.

## Запуск

```bash
cp .env.example .env            # вписать HF_TOKEN (нужен pyannote/segmentation-3.0)
mkdir -p audio
cp /путь/к/записи.mp3 audio/call.mp3

docker compose run --rm trial /audio/call.mp3 v3_rnnt
```

Прогнать стоит на стерео- и моно-записи, сравнив варианты модели:
`v3_rnnt` (точность), `v3_ctc` (скорость), `v3_e2e_rnnt` (с пунктуацией).

## Как читать результат

- `RTF` — секунд счёта на секунду аудио. `RTF<1` быстрее реалтайма; `RTF=2` —
  на 10-минутный звонок ~20 минут счёта (при лимите в 3 потока).
- Транскрипт в выводе сравнить по-русски с whisper-транскриптом того же
  звонка — это оценка качества.

Если RTF плохой даже на 3 потоках — следующий рычаг ONNX-экспорт (`model.to_onnx`),
до того как писать сервис.

## Снос стенда (чтобы не остались веса-сироты ~0.5+ ГБ)

`docker compose down` НЕ удаляет `cache/` — это bind-mount, а не named volume
(даже `down -v` его не трогает). Веса надо убирать явно:

```bash
docker compose down --rmi local --remove-orphans   # контейнеры + локальные образы
rm -rf cache/                                       # скачанные веса GigaAM + pyannote
docker builder prune -f                             # (опц.) build-кэш
```

Чистовой (прод) враппер кэширует веса в СВОЁМ именованном volume, не в этом
`cache/`, чтобы экспериментальные и боевые веса не пересекались.

## Прим.

- CPU-only, GPU на машине нет. torch ставится из CPU-индекса.
- `cache/` и `audio/` и `.env` в `.gitignore` — веса и записи в репозиторий не
  попадают.
- Лимит потоков задаётся `GIGAAM_THREADS` (compose) + `cpus` как страховка docker.
